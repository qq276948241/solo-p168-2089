# Roguelike 地牢游戏

一个用 Go 写的终端 Roguelike 小游戏。8×8 随机地图，踩格子探索，回合制战斗，通关后可连续挑战下一关。

---

## 游戏玩法

每局游戏随机生成一张 8×8 的格子地图。你从左上角 (0,0) 出发，每次移动到相邻的上下左右格子，踩到未知格子会触发不同事件。

**通关条件**：清掉地图上所有的怪物格。

**失败条件**：角色生命值归零。

一关通关后可以进入下一关，怪物数量会逐关递增。

## 快速开始

```bash
# 编译
go build -o roguelike.exe .

# 运行
.\roguelike.exe

# 或者直接跑
go run .
```

终端需要支持 UTF-8 显示字符（Windows 10+ 的 PowerShell 或 Windows Terminal 都可以）。如果不支持 ANSI 颜色会自动降级为纯文本。

## 地图格子

地图上的格子有 6 种类型，未探索时显示为 `?`。

| 符号 | 类型 | 说明 |
|------|------|------|
| `@` | 玩家 | 你所在的位置 |
| `·` | 空地 | 什么都没有，安全通过 |
| `✗` | 怪物格 | 踩到进入回合制战斗 |
| `□` | 宝箱 | 回血 + 永久加攻击力 |
| `~` | 陷阱 | 踩到扣血 |
| `≈` | 泉水 | 踩到满血。**满血踩到不消耗，可留到以后用** |

### 格子数值

**宝箱**：
- 恢复 15~29 点生命值（不超过上限）
- 攻击力永久 +2~+4

**陷阱**：
- 扣除 8~15 点生命值

**泉水**：
- 直接恢复到满血
- 满血踩到不消耗格子，下次路过还能再用

## 回合制战斗

踩到怪物格进入战斗界面。每回合你先行动，然后怪物行动。

### 战斗操作

**1. 攻击**

造成 `ATK + (-1~+2)` 点伤害。如果上回合放了「重击」buff，伤害翻倍。

**2. 防御**

不造成伤害，但本回合受到的怪物伤害减半（和自身防御叠加后再减半）。

**3. 技能**

打开技能菜单，选择一个技能释放。技能有冷却回合数，CD 中无法使用。详细说明见下方「技能系统」。

### 怪物狂暴

怪物生命值低于 50% 时，有 30% 概率进入**狂暴状态**：
- 攻击力翻倍
- 持续 2 回合
- 狂暴状态的怪物 ASCII 画会显示为红色

## 角色属性

初始属性（每局新游戏从这里开始）：

| 属性 | 初始值 | 说明 |
|------|--------|------|
| HP | 50 / 50 | 生命值，归零则阵亡 |
| ATK | 10 | 攻击力，决定普通攻击伤害 |
| DEF | 5 | 防御力，减少怪物伤害 |
| 金币 | 0 | 击败怪物获得，目前用于统计 |
| 杀怪数 | 0 | 本关击杀怪物数量 |
| 关卡 | 1 | 当前所在关数 |

> 关卡之间属性**继承**，不会重置。血量低了记得找泉水或宝箱回血。

## 技能系统

开局自带 3 个技能，每个技能独立冷却。

### 技能列表

| 技能 | 效果 | 冷却 |
|------|------|------|
| **重击** | 给自己加一个 buff，**下一次普通攻击伤害 ×2**。如果开了重击 buff 又放技能没用掉，buff 不会消失，会一直等到你普攻为止 | 3 回合 |
| **治疗** | 立即恢复 20 点生命值（不超过上限） | 4 回合 |
| **破甲** | 立即造成 `ATK + (-1~+2)` 点**真实伤害**，无视怪物防御。这是唯一一个能在怪物回合之前就让它掉血的技能 | 3 回合 |

### 冷却规则

- 技能释放成功后进入冷却
- **每回合开始自动 -1**（玩家回合之前就扣）
- 冷却 0 表示可以使用
- 技能菜单里每个技能后面会显示 `[CD:N]`

## 怪物种类

| 名字 | 说明 |
|------|------|
| 史莱姆 | 最弱的怪，血少攻低 |
| 哥布林 | 中等强度 |
| 骷髅兵 | 中等偏上 |
| 蝙蝠群 | 波动大 |
| 狼人 | 最强的小怪，攻击力高 |

怪物强度随机在 3 个等级内波动，同一种怪物可能属性差很多。

## 关卡与评价

| 关卡 | 怪物数量 |
|------|----------|
| 1 | 8 |
| 2 | 10 |
| 3 | 12 |
| N | 8 + (N-1) × 2 |

每关通关后会根据**杀怪数 + 剩余血量**给出评价等级：

| 等级 | 条件 |
|------|------|
| S | 分数 ≥ 120 |
| A | 分数 ≥ 90 |
| B | 分数 ≥ 60 |
| C | 分数 ≥ 30 |
| D | 分数 < 30 |

> 评分公式：`杀怪数 × 10 + (剩余HP / 最大HP) × 50`

## 代码结构

```
project168/
├── main.go          主入口 + 游戏主循环 + 地图/渲染
├── combat.go        战斗逻辑抽离（回合、技能菜单、战斗UI）
├── skill.go         技能系统（Skill接口、SkillManager、3个技能实现）
├── encounter.go     遭遇系统（怪物狂暴、泉水、关卡难度、评价、怪物画）
├── go.mod
└── roguelike.exe    （编译产物）
```

### 各文件职责

**main.go**
- `main()`：程序入口，控制多周目循环
- `runLevel()`：单关主循环，移动、踩格子事件分发
- `generateMap()`：随机地图生成，按关卡放不同数量怪物
- `printStatus()` / `printMap()`：顶部状态栏 + 地图渲染
- `detectANSI()` / `color()`：ANSI 颜色检测+降级

**combat.go**
- `Combat()`：完整的战斗回合循环（玩家行动→怪物行动→下回合）
- `showSkillMenu()`：技能选择子菜单
- `printCombatStatus()`：战斗界面顶部状态栏 + HP条

**skill.go**
- `Skill` 接口：所有技能的统一抽象（`ID/Name/Description/Cooldown/Execute`）
- `HeavySkill` / `HealSkill` / `PierceSkill`：3 个技能实现
- `SkillManager`：技能注册、冷却管理、`Use(id)` 统一调度

**encounter.go**
- `MonsterEx`：怪物扩展（狂暴逻辑，虽然现在 Monster 结构体已经直接带 Rage 字段了）
- `DrawMonsterRage()`：怪物 ASCII 画，狂暴时变红色
- `UseFountain()`：泉水效果
- `GetMonsterCount()`：按关卡返回怪物数量
- `CalculateGrade()` / `PrintGrade()`：通关评价计算 + 显示

## 开发者笔记

### 加一个新技能

只需要改 [skill.go](file:///d:/code/ai-prompt/solo-chrome-dev-F12/repos/repo168/project168/skill.go)：

1. 写一个结构体实现 `Skill` 接口的 5 个方法（通常内嵌 `BaseSkill` 省得写重复代码）
2. 在 `NewSkillManager()` 里 `sm.Register(NewXXXSkill())` 注册一下
3. 在 `SkillManager.SkillIDs()` 返回列表里加上新技能的 ID（决定菜单显示顺序）

不需要改 combat.go 和 main.go，其他地方会自动适配。

### 加一个新怪物

在 [encounter.go](file:///d:/code/ai-prompt/solo-chrome-dev-F12/repos/repo168/project168/encounter.go) 的 `DrawMonsterRage()` 里加一张 ASCII 画，然后在 [main.go](file:///d:/code/ai-prompt/solo-chrome-dev-F12/repos/repo168/project168/main.go) 的 `generateMonster()` 名字列表里加进去就行。

### 调数值

所有可调的常量集中在：

| 位置 | 内容 |
|------|------|
| `skill.go` 顶部 `const` 块 | 技能名字/描述/CD/数值 |
| `main.go` 的 `MapSize` | 地图大小（默认 8） |
| `main.go` 的 `generateMap()` | 每关地图放几个宝箱/陷阱/泉水 |
| `encounter.go` 的 `GetMonsterCount()` | 关卡怪物递增公式 |
| `encounter.go` 的 `CalculateGrade()` | 评价分数线 |
| `main.go` 的 `generateMonster()` | 怪物随机强度区间 |
| `main.go` 的 `openChest()` / `triggerTrap()` | 宝箱/陷阱数值 |

### 数据驱动思路

技能和怪物如果以后要扩展，推荐方向是把 `Skills` 常量和怪物列表改成从 JSON/YAML 读入，游戏启动时 `Unmarshal` 后动态注册。目前用 `const` 块已经够小项目用了，后续想加数据驱动可以直接在 `skill.go` 的 `NewSkillManager()` 里把构造替换成读配置文件。

### 已知可以改进的点

- 怪物目前没有真正的「防御」属性，破甲技能意义是不扣防御减伤，可以考虑给怪物加 DEF
- 金币目前只能看，可以加商店系统（比如在泉水格子买道具）
- 地图可以做连通性保证，目前是纯随机，极端情况可能被陷阱/怪物围住但概率很低
- 存档/读档：序列化 Player + Map 状态到文件即可
